Create a blank project

Switch to newly created directory. Use pelican-quickstart to create project skeleton. This command will help you create a new Pelican-based website.

It will ask you questions on how you would like to configure your website. Once you have answered these questions it will create the project skeleton accordingly.

To keep it simple, I will be answering No to most of the questions. We will looks at more advanced examples later.

Generate site

Navigate back to your site directory and run pelican to generate your website.

SarangMac:jaiswalsarang.github.io sarangjaiswal$ pelican -s pelicanconf.py content/ -o output/
Done: Processed 1 article, 0 drafts, 0 pages, 0 hidden pages and 0 draft pages in 0.15 seconds.

See below for details on arguments of pelican command

  -o OUTPUT, --output OUTPUT
                        Where to output the generated files. If not specified,
                        a directory will be created, named "output" in the
                        current path. (default: None)
  -s SETTINGS, --settings SETTINGS
                        The settings of the application, this is automatically
                        set to pelicanconf.py if a file exists with this name.
                        (default: None)

Above command will convert your Markdown (.md) file into html. All the converted files will be placed in output directory.

Preview site

Navigate back to the parent folder of the site and run 

SarangMac:jaiswalsarang.github.io sarangjaiswal$ pelican -l content/ -o output/ -s pelicanconf.py -p 8085

See below for details on arguments of pelican command

  -t THEME, --theme-path THEME
                        Path where to find the theme templates. If not
                        specified, it will use the default one included with
                        pelican. (default: None)
  -o OUTPUT, --output OUTPUT
                        Where to output the generated files. If not specified,
                        a directory will be created, named "output" in the
                        current path. (default: None)
  -s SETTINGS, --settings SETTINGS
                        The settings of the application, this is automatically
                        set to pelicanconf.py if a file exists with this name.
                        (default: None)

  -p PORT, --port PORT  Port to serve HTTP files at. (default: 8000) (default:
                        None)

Issues

Below were few issues which I came across while creating this article. I have listed the issue and the solution for each of them below

If the live website looks something like below then there is some issue with the theme css file.

Make following changes to the Makefile

Original:

publish:
    $(PELICAN) $(INPUTDIR) -o $(OUTPUTDIR) -s $(PUBLISHCONF) $(PELICANOPTS)

Change to:

publish:
    $(PELICAN) $(INPUTDIR) -o $(OUTPUTDIR) -s $(CONFFILE) $(PELICANOPTS)
If you came across the below The requested URL returned error: 403 error then refer the Stack Overflow here to resolve it.
